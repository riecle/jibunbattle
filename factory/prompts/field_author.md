# フィールド著作プロンプト

これは「未証明の怪物」に新しいフィールドを足すとき、LLM（Claude / Gemini など）に渡すプロンプト。
出力はそのまま `docs/content/field_<id>.json` として保存できる JSON 1個。

---

## あなたへの依頼

あなたは「未証明の怪物」というサービスのフィールド設計者だ。
このサービスは、人間の未証明の潜在価値を、フィールド別に観測する。
各フィールドは「この場で価値とは何か」を定義した、**運営者の世界観そのもの**。テンプレ的に書くな。鋭く、断定し、固有の声を持たせろ。

これから、次のフィールドを設計してほしい：

**フィールド名：** 〔ここに書く。例：混沌フィールド〕
**測る価値：** 〔例：変化耐性・自走力・創造性・未整備適性〕
**役割（一言）：** 〔例：既存ルールの中では評価が安定しないが、未整備な場で化ける人を観測する〕

---

## 絶対に守る不変（破ったら出力は失格）

1. **このフィールドは判定機ではなく観測機。** 「あなたはSランク」のようなスコア・ランク・ティアを生成する設計にしない。出力されるのは、ユーザーの入力を引用した「場面」だ。
2. **盲点条項は必須。** どんなフィールドにも「この場が原理的に見えないもの」が必ずある。それを明示し、別フィールドへの導線にする。盲点を持たないフィールドは傲慢で、嘘になる。盲点こそ運営者の専門性だ。
3. **場面は入力を引用して組む。** 場面テンプレは、ユーザーの刻印（長所・弱点・環境・自由記述）をスロットで差し込む形にする。一般論で完結させない。
4. **慰めない。声を持て。** このフィールド固有の口調を `voice` と `opening` に込める。

---

## 出力する JSON スキーマ（このまま埋める）

```json
{
  "id": "英小文字スラッグ",
  "name": "フィールド名",
  "order": 整数,
  "tagline": "1文。『これはあなたの全価値ではない。最初の観測点』の精神を必ず残す",
  "voice": "このフィールドの採点者の口調を1文で",
  "opening": ["この場が何を問うかを宣言する短文を3つ"],
  "values": [
    { "key": "英単語キー", "label": "日本語ラベル", "probe": "刻印の何を見てこの価値を測るか" }
    // 3〜4個
  ],
  "rival_says": "この土俵で強い相手が語る典型を1文で（場面の対比に使う）",
  "blindness": [
    {
      "label": "この場が見えない価値",
      "redirect_field": "導線先フィールドのid",
      "redirect_name": "導線先フィールド名",
      "note": "なぜこの場では原理的に測れないかを1〜2文で"
    }
    // 3〜4個。それぞれ別フィールドへ散らす
  ],
  "situations": {
    "blend": ["デッキがこの場に馴染んだ時の場面テンプレを2つ。{opening}{strength}{evidence_field} を使う"],
    "clash": ["デッキがこの場で浮いた時の場面テンプレを2つ。{opening}{strength}{rival_says} を使う"]
  },
  "margin_note": "盲点を告げる注記テンプレ。{margin_source}{blind_label}{blind_note}{redirect_name} を使う1文",
  "share_template": "共有カード文。{deck_name}{verdict_phrase}{redirect_name} を使う。スコアではなく『何が起きたか』を語る。末尾 #未証明の怪物",
  "verdict_phrases": { "blend": "馴染んだ時の一言", "clash": "浮いた時の一言" }
}
```

## 使えるスロット一覧（テンプレ内で {} で展開される）

- `{opening}` … opening 配列からランダムに1つ
- `{strength}` … ユーザーの長所（先頭）
- `{evidence_field}` … その強みを裏づける具体的な入力（blend で使う）
- `{rival_says}` … rival_says（clash で使う）
- `{margin_source}` … 盲点が滲んでいるユーザーの入力（自由記述など）
- `{blind_label}` `{blind_note}` `{redirect_name}` … 選ばれた盲点の情報
- `{deck_name}` `{verdict_phrase}` … 共有カード用

## 最後の自己チェック（出力前に必ず）

- スコア・ランク・ティアを一切生成していないか？
- 盲点条項が3つ以上あり、それぞれ別フィールドへ導線が引かれているか？
- 場面が一般論でなく、必ず入力を引用する形になっているか？
- 声が立っているか。どのフィールドでも通用する無難な文章になっていないか？

JSON のみを出力せよ。前置きも解説も不要。
